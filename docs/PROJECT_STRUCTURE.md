## Module Roles & Responsibilities

### `rag/` — Pipeline Core

| File | Role | Key Details |
|---|---|---|
| `embeddings.py` | Embedding model factory | `sentence-transformers/all-MiniLM-L6-v2` via `langchain-huggingface` |
| `ingestion.py` | PDF → chunks → ChromaDB | chunk_size=500, overlap=50; attaches `role_allowed`, `department`, `source` metadata |
| `llm.py` | LLM wrapper | Groq API, `llama-3.1-8b-instant`, temperature=0, strict "answer from context only" prompt |
| `retriever.py` | Similarity search | Top-5 docs (`k=5`) via Chroma `.as_retriever()` |
| `vector_store.py` | DB create/load | Persists to `vector_db/` directory (git-ignored) |

### `rbac/` — Access Control

| File | Role | Key Details |
|---|---|---|
| `access_control.py` | Document-level filter | Reads `role_allowed` list from chunk metadata; `"all"` bypasses restrictions; case-insensitive |

### `services/` — Orchestration

| File | Role | Key Details |
|---|---|---|
| `rag_service.py` | Pipeline glue | retrieve → rbac_filter → guard (empty = "No information available") → llm |

## Data Flow

```
User Input (query + role)
        │
        ▼
  get_relevant_docs(query)          ← rag/retriever.py
        │  [top-5 semantic matches from ChromaDB]
        ▼
  rbac_filter(docs, user_role)      ← rbac/access_control.py
        │  [only docs where role_allowed contains user_role or "all"]
        ▼
  [empty? → "No information available"]
        │
        ▼
  get_llm_response(query, docs)     ← rag/llm.py
        │  [Groq Llama 3.1, strict context-only prompt]
        ▼
  Response printed to user          ← app.py
```
